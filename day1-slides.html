Part 1: Git Basics

What is Version Control?

A system to track changes in files over time.

Why needed? Collaboration, backup, history, avoid chaos!

Meet Git

  • Created by Linus Torvalds in 2005 for Linux
  • Distributed VCS: Fast, branching, merging
  • Works locally on your machine

Meet GitHub

  • Launched in 2008 as Git hosting platform
  • Web-based: Collaboration, issue tracking, social coding
  • Home to millions of open-source projects

Why Git & GitHub Matter

  • Essential for modern software (Linux, React, etc.)
  • Global collaboration: Contribute to any project
  • Prevents data loss, builds portfolios

Git + GitHub = Magic

Git handles local changes; GitHub syncs remotely for teamwork.

Local Repo (Git) <--> Remote Repo (GitHub) | | Commits Pull Requests Branches Issues Collaboration!

Key Concepts

  • Repository: Project storage
  • Commits: Change snapshots
  • Branches: Parallel work lines
  • Pull Requests: Review & merge changes

Git Workflow

1. Clone repo: git clone https://github.com/GitHer-Muna/Git-GitHub-Workshop.git 2. Create branch: git checkout -b feature 3. Make changes & commit: git add . && git commit -m "msg" 4. Push branch: git push origin feature 5. Create PR on GitHub 6. Merge after review
Part 1: Git Basics

Installation

  • Git: Download from git-scm.com
  • GitHub: Sign up at github.com

Setup Git

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Use SSH keys or tokens for secure access.
